# Hitokoto Offline merger (一言离线处理与整合)

> 核心目标：摆脱对第三方 API 的依赖，将一言（Hitokoto）句子库本地化部署，实现网站“零延迟”加载随机句子。

本项目包含一套 Python 脚本和自动化流程，用于将 Hitokoto 的原始分类 JSON 文件（如 `a.json`, `b.json` 等）清洗、去重并合并为一个极致精简的 `json` 文件，方便静态网站（如 MkDocs, Hexo, Hugo）直接调用。

## 问题背景

通常我们在网站上展示“一言”时，会调用官方 API。但在以下场景中，**本地化部署**是更好的选择：

* **速度优先**：无需等待 API 响应，利用 CDN (如 Cloudflare) 缓存 JSON，加载速度极快。
* **高可用性**：即使 API 服务宕机或网络波动，你的网站句子依然能正常展示。
* **完全掌控**：你可以自由删除不喜欢的句子，或混入自己收集的语录，无需审核。
